# v2.1 — Monorepo refactor

**Goal:** Split the current single package into two independent packages
(`mva-core`, `mva-cli`) inside a `uv` workspace, so that non-CLI consumers
(web UI, GUI, scripts) can `pip install mva-core` without pulling in
prompt-toolkit or rich.

---

## Current structure (v2.0 — actual state)

The internal restructuring (tools/skills moved under `agent/`, `utils/` split) is
already complete.  The live layout is:

```
src/mva/
├── __init__.py          → main() entry point, delegates to cli.app()
├── __main__.py          → `python -m mva`
├── config.py            → model.yaml loader
├── agent/               → self-contained agent package
│   ├── __init__.py      → re-exports LLMClient, Session, ToolDef, execute_tool, SkillDef, …
│   ├── client.py        → OpenAI-compatible HTTP client
│   ├── session.py       → Session (history, tool-calling loop)
│   ├── types.py         → ChatMessage, StreamingDelta, CompletionUsage, LLMError
│   ├── _system.py       → signal handling, system prompt building, message construction
│   ├── tools/           → Tool system (ToolDef, Tool, ToolResult, registry, builtins)
│   │   ├── __init__.py
│   │   ├── base.py
│   │   ├── registry.py
│   │   ├── path_security.py
│   │   └── builtin/     → bash, edit, fetch_url, list_files, read, write
│   └── skills/
│       └── __init__.py  → SkillDef, discover_skills, build_skills_prompt
├── cli/                 → CLI/UI layer (consumes agent)
│   ├── __init__.py
│   ├── app.py           → Typer entry point
│   ├── console.py       → prompt-toolkit session, completer, toolbar
│   ├── renderer.py      → streaming event renderer, markdown renderer, spinner
│   ├── repl.py          → REPL loop
│   └── _commands.py     → command dispatch, display helpers, session persistence
└── utils/               ← **DELETED** — split into agent/_system.py + cli/_commands.py
```

## Execution order

### Step 1: Create `packages/mva-core/` tree

```
mkdir -p packages/mva-core/src/mva_core/{agent,tools/builtin,skills}
```

1. Copy files from `src/mva/agent/` → `packages/mva-core/src/mva_core/agent/`
2. Copy files from `src/mva/agent/tools/` → `packages/mva-core/src/mva_core/tools/`
3. Copy files from `src/mva/agent/skills/` → `packages/mva-core/src/mva_core/skills/`
4. Copy `src/mva/config.py` → `packages/mva-core/src/mva_core/config.py`
5. Copy `src/mva/agent/_system.py` → `packages/mva-core/src/mva_core/_system.py`
6. Create `packages/mva-core/src/mva_core/__init__.py` with flat re-exports
7. Update imports in all copied files (`mva.` → `mva_core.`)
8. Write `packages/mva-core/pyproject.toml`

### Step 2: Create `packages/mva-cli/` tree

```
mkdir -p packages/mva-cli/src/mva_cli
```

1. Copy files from `src/mva/cli/` → `packages/mva-cli/src/mva_cli/`
2. Update imports to reference `mva_core.*` instead of `mva.*`
3. Write `packages/mva-cli/pyproject.toml` (depends on `mva-core`)

### Step 3: Update root `pyproject.toml`

- Convert to workspace root with `[tool.uv.workspace]`
- Add `members = ["packages/*"]`
- Remove direct dependencies (they move to the sub-packages)
- Keep a thin `mva` shim package if desired (or remove entirely)

### Step 4: Create shim `src/mva/` (optional)

If we want `python -m mva` and `mva` CLI command to keep working
without re-installing, create thin re-exports:

```python
# src/mva/__init__.py
from mva_cli import app  # noqa: F401
main = app  # for `mva:main` entry point
```

```python
# src/mva/__main__.py
from mva_cli.app import app
app()
```

Or simply update `pyproject.toml` to point `[project.scripts]` at
`mva_cli` directly.

### Step 5: Delete old single-package code

After testing, remove `src/mva/agent/`, `src/mva/cli/`, `src/mva/config.py`.

### Step 6: Test

```bash
uv sync
uv run --package mva-core python -c "from mva_core import Session; print('mva-core OK')"
uv run --package mva-cli python -c "from mva_cli import app; print('mva-cli OK')"
uv run mva --help
uv run mva --print "hello"
```

### Step 7: Remove stale `__pycache__` dirs

```bash
find . -type d -name __pycache__ -exec rm -rf {} + 2>/dev/null || true
```
