# v2.1 — Monorepo refactor

**Goal:** Split the current single package into two independent packages
(`mva-core`, `mva-cli`) inside a `uv` workspace, so that non-CLI consumers
(web UI, GUI, scripts) can `pip install mva-core` without pulling in
prompt-toolkit or rich.

---

## Current structure (v2.0)

```
src/mva/
├── agent/          ← LLM client + Session (history, tool loop)
├── cli/            ← Typer app, prompt-toolkit, renderer
├── tools/          ← tool definitions, registry, builtins
├── skills/         ← skill discovery and prompt building
├── config.py       ← model.yaml loader
├── utils/          ← mixed: signal handling, system prompt, messages, commands, display
├── __init__.py     ← re-exports `_app` from cli
└── __main__.py     ← `python -m mva`
```

### Problem:
`utils/__init__.py` mixes core concerns (signal handling, system prompt building,
message construction) with CLI concerns (command dispatch, display helpers).
A web UI cannot reuse `Session` without importing rich/typer.

## What moves where

### From `mva/agent/` → `mva-core/mva_core/agent/`
Entire directory: `__init__.py`, `client.py`, `session.py`.

Import changes inside `session.py`:
- `from mva.tools import execute_tool, ToolResult` → `from mva_core.tools import execute_tool, ToolResult`
- `from mva.utils import build_messages` → `from mva_core._system import build_messages`
- `from mva.utils import is_cancel_requested` → `from mva_core._system import is_cancel_requested`

Import changes inside `client.py`:
- `from mva.config import ConfigError, load_config` → `from mva_core.config import ConfigError, load_config`
- `from mva.utils import is_cancel_requested` → `from mva_core._system import is_cancel_requested`

### From `mva/tools/` → `mva-core/mva_core/tools/`
Entire directory (no structural changes).

Import changes:
- `from mva.agent import ToolDef` → `from mva_core.agent import ToolDef`

### From `mva/skills/` → `mva-core/mva_core/skills/`
Entire directory (no structural changes).

Import changes:
- (no internal imports to change — skills is self-contained)

### From `mva/config.py` → `mva-core/mva_core/config.py`
Single file, no import changes needed.

### From `mva/utils/__init__.py` → split

**→ `mva-core/mva_core/_system.py`:**
- `_streaming_active`, `_cancel_requested`, `_mark_streaming_start()`,
  `_mark_streaming_stop()`, `is_cancel_requested()`, `install_signal_handler()`,
  `_signal_handler()`
- `DEFAULT_SYSTEM_PROMPT`

  `_load_append_system_md()`, `_load_agent_md()`
- `build_system_prompt()`
- `build_messages()`

**→ `mva-cli/mva_cli/_commands.py`:**
- `print_header()`, `goodbye()`, `print_help()`
- `handle_command()`
- `_print_tools()`, `_print_skills()`, `_toggle_skill()`
- `_print_model_info()`, `_switch_model()`, `_switch_provider()`, `_list_providers()`

### `mva/cli/` → `mva-cli/mva_cli/`
Entire directory, but update import paths.

## Dependency graph (after)

```
mva-cli ──→ mva-core
              ├── agent/ ──→ config, _system
              ├── tools/  ──→ agent (ToolDef)
              ├── skills/ ── (pure)
              ├── config.py
              └── _system.py
```

`mva-core` has **zero** external Python dependencies (only stdlib +
`pyyaml` + `requests`).  No `rich`, no `typer`, no `prompt-toolkit`.

---

## Execution order

1. Create `packages/mva-core/` tree — copy directories, create `_system.py`, write `__init__.py` + `pyproject.toml`
2. Create `packages/mva-cli/` tree — copy directories, create `_commands.py`, update import paths, write `pyproject.toml`
3. Update root `pyproject.toml` — add `[tool.uv.workspace]` and workspace members
4. Delete old `src/mva/` (everything migrated)
5. Update `mva/__init__.py` + `mva/__main__.py` to delegate to `mva-cli`
6. Test: `uv sync && uv run mva --help && uv run mva --print "hello"`
7. Remove stale `__pycache__` dirs
